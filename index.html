01 / Introduction

What is SQLite?

A powerful, embedded relational database — no server, no setup, just a file on disk.

🗄️

Self-Contained

SQLite is a C library that implements a complete SQL database engine. Everything is stored in a single cross-platform file — no external processes needed.

⚡

Serverless

Unlike PostgreSQL or MySQL, SQLite requires no server process. The application reads and writes directly to the database file, making it ultra-fast for local data.

🔓

Public Domain

SQLite is completely free — in the public domain. You can use it in any project, commercial or personal, without any restrictions or licensing fees.

📌 Simple Definition

SQLite is an open-source, embedded relational database management system (RDBMS) that stores data in structured tables using SQL (Structured Query Language). Unlike traditional databases, it does not run as a separate server — it is a library that is linked into your application and reads/writes directly to a single file on disk.

02 / Setup & Basics

How to Use SQLite

Installation is trivial — SQLite is already bundled with Python, Android, and most operating systems.

01

Installation

SQLite is pre-installed on macOS and Linux. On Windows, download the CLI from the official website. In Python it's built-in — no pip needed!

Terminal — macOS/Linux 
# Check if already installed
sqlite3 --version

# Ubuntu / Debian
sudo apt-get install sqlite3

# macOS (already installed, or via Homebrew)
brew install sqlite
Python — Built-in 
# No installation needed!
import sqlite3

# Check Python sqlite version
print(sqlite3.sqlite_version)
# Output: 3.45.3
02

Create a Database

Creating a database is as simple as opening a file. SQLite creates it automatically if it doesn't exist.

CLI 
# Opens (or creates) mydb.sqlite
sqlite3 mydb.sqlite

# You'll see the prompt:
SQLite version 3.45.3
Enter ".help" for usage hints.
sqlite>
Python 
import sqlite3

# Creates file if it doesn't exist
conn = sqlite3.connect("mydb.sqlite")
cursor = conn.cursor()

# Use in-memory database (temp)
conn = sqlite3.connect(":memory:")

Essential CLI Commands

04

Create Your First Table

Tables define the structure of your data — think of them like spreadsheet columns with data types enforced.

SQL 
CREATE TABLE students (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    name       TEXT    NOT NULL,
    age        INTEGER,
    email      TEXT    UNIQUE,
    grade      REAL    DEFAULT 0.0,
    enrolled   TEXT    DEFAULT '2024-01-01'
);

-- SQLite data types:
-- TEXT (strings), INTEGER (whole numbers)
-- REAL (decimals), BLOB (binary), NULL
03 / CRUD Operations

Create, Read, Update, Delete

The four fundamental database operations — every app you build will use these constantly.

INSERT

Creating Records

The INSERT INTO statement adds new rows to a table. You can insert a single row or multiple rows at once.

Insert Single Row 
-- Insert one student
INSERT INTO students (name, age, email, grade)
VALUES ('Emma Davis', 23, 'emma@uni.edu', 88.5);

-- ID is auto-generated (AUTOINCREMENT)
-- Omit columns that have defaults
Insert Multiple Rows 
-- Insert multiple students at once
INSERT INTO students (name, age, email, grade)
VALUES
  ('Frank Moore',  20, 'frank@uni.edu',  91.0),
  ('Grace Hall',   22, 'grace@uni.edu',  79.5),
  ('Henry King',   24, 'henry@uni.edu',  84.2);

-- Much faster than 3 separate INSERTs
SELECT

Reading Records

The SELECT statement retrieves data. It's the most powerful and flexible operation — you can filter, sort, join, group, and aggregate.

Basic Queries 
-- Get all students
SELECT * FROM students;

-- Get specific columns
SELECT name, grade FROM students;

-- Filter with WHERE
SELECT * FROM students
WHERE grade > 90;

-- Sort results
SELECT * FROM students
ORDER BY grade DESC;
Advanced Queries 
-- Aggregate functions
SELECT
  COUNT(*) AS total,
  AVG(grade) AS avg_grade,
  MAX(grade) AS highest
FROM students;

-- Pattern matching
SELECT * FROM students
WHERE name LIKE 'A%';

-- Limit results
SELECT * FROM students
ORDER BY grade DESC LIMIT 3;
UPDATE

Updating Records

The UPDATE statement modifies existing rows. Always use WHERE — without it, every row gets updated!

Update One Record 
-- Update Bob's grade
UPDATE students
SET grade = 90.0
WHERE id = 2;

-- Update multiple columns
UPDATE students
SET
  grade = 95.0,
  age   = 23
WHERE email = 'alice@uni.edu';
Conditional Updates 
-- Add 5 points to all grades below 80
UPDATE students
SET grade = grade + 5
WHERE grade < 80;

-- ⚠️ DANGER — updates ALL rows!
UPDATE students
SET grade = 0;  -- No WHERE!

-- Always check your WHERE clause
DELETE

Deleting Records

The DELETE statement removes rows permanently. Like UPDATE, always use WHERE — omitting it wipes the entire table.

Delete Specific Rows 
-- Delete a student by ID
DELETE FROM students
WHERE id = 4;

-- Delete students with low grades
DELETE FROM students
WHERE grade < 60;

-- Delete by name pattern
DELETE FROM students
WHERE name = 'Bob Smith';
Drop vs Delete 
-- Delete ALL rows (keeps structure)
DELETE FROM students;

-- Faster way to clear everything
DELETE FROM students;
DELETE FROM sqlite_sequence
WHERE name = 'students'; -- reset IDs

-- Remove table entirely
DROP TABLE students;       -- gone!

05 / Evaluation

Advantages & Disadvantages

Understanding when SQLite shines — and when to use something else.

✅ Advantages

⚡

Lightweight & Fast

The entire library is ~600KB. No process overhead — reads and writes go directly to disk with minimal latency for small-medium datasets.

🖥️

Serverless Architecture

No installation, configuration, or running daemon required. Just open a file and start querying — perfect for embedded applications.

📦

Portable Single File

Your entire database is one file you can copy, email, or version-control. Works identically on Windows, Linux, macOS, iOS, Android.

🔧

Zero Configuration

No user accounts, no network settings, no permission schemas. Just create a connection and start working in seconds.

🤝

ACID Compliant

Fully supports transactions (Atomicity, Consistency, Isolation, Durability) — your data stays safe even if the app crashes mid-write.

⚠️ Disadvantages

👥

Limited Concurrency

SQLite uses file-level locking. Only one writer at a time is allowed — poor choice for apps with many simultaneous users writing data.

📈

Not for Large Scale

Performance degrades with very large datasets (hundreds of GB) or high-throughput write workloads. Use PostgreSQL or MySQL instead.

🌐

No Network Access

SQLite is file-based and cannot be accessed over a network. Every client must have local access to the database file.

🔒

Limited Security

No built-in user authentication or row-level security. Anyone with file system access can read the database file directly.

🔄

Partial SQL Support

Some advanced SQL features (like RIGHT JOIN, complete ALTER TABLE, stored procedures) are missing or limited compared to full RDBMS systems.

SQLite with Popular Languages

Connect SQLite to any language or framework — it takes just a few lines of code to get started.

🐍

Python + SQLite3

Built into Python's standard library — no installation needed. Uses the sqlite3 module.

✓ Built-in — no pip required
Connect & Create Table 
import sqlite3

# Connect (creates file if not exists)
conn = sqlite3.connect("mydb.sqlite")
cur  = conn.cursor()

# Create table
cur.execute("""
  CREATE TABLE IF NOT EXISTS users (
    id    INTEGER PRIMARY KEY AUTOINCREMENT,
    name  TEXT NOT NULL,
    email TEXT UNIQUE
  )
""")
conn.commit()
print("Table ready!")
CRUD Operations 
# INSERT
cur.execute("INSERT INTO users (name, email) VALUES (?, ?)",
           ("Alice", "alice@example.com"))
conn.commit()

# SELECT
cur.execute("SELECT * FROM users")
rows = cur.fetchall()
for row in rows:
    print(row)

# UPDATE
cur.execute("UPDATE users SET name=? WHERE id=?", ("Bob", 1))

# DELETE
cur.execute("DELETE FROM users WHERE id=?", (1,))
conn.commit()
conn.close()

PHP + PDO / SQLite3

PHP has two ways to connect: the SQLite3 class and the more flexible PDO abstraction layer.

extension=pdo_sqlite ← enable in php.ini
PDO Connection 
<?php
try {
    // Connect via PDO
    $pdo = new PDO("sqlite:mydb.sqlite");
    $pdo->setAttribute(
        PDO::ATTR_ERRMODE,
        PDO::ERRMODE_EXCEPTION
    );

    // Create table
    $pdo->exec("CREATE TABLE IF NOT EXISTS users (
        id    INTEGER PRIMARY KEY AUTOINCREMENT,
        name  TEXT NOT NULL,
        email TEXT UNIQUE
    )");

    echo "Connected successfully!";
} catch (PDOException $e) {
    echo "Error: " . $e->getMessage();
}
Insert & Select (PDO) 
// INSERT with prepared statement
$stmt = $pdo->prepare(
    "INSERT INTO users (name, email) VALUES (?, ?)"
);
$stmt->execute(['Alice', 'alice@mail.com']);

// SELECT all rows
$stmt = $pdo->query("SELECT * FROM users");
while ($row = $stmt->fetch(PDO::FETCH_ASSOC)) {
    echo $row['name'] . ' — ' . $row['email'] . "\n";
}

// UPDATE
$pdo->prepare("UPDATE users SET name=? WHERE id=?")
    ->execute(['Bob', 1]);

Connect & Query 
import java.sql.*;

public class Main {
  public static void main(String[] args) throws Exception {
    String url = "jdbc:sqlite:mydb.sqlite";

    try (Connection con = DriverManager.getConnection(url);
         Statement st  = con.createStatement()) {

      // Create
      st.execute("CREATE TABLE IF NOT EXISTS users " +
                 "(id INTEGER PRIMARY KEY, name TEXT)");

      // Insert
      st.execute("INSERT INTO users VALUES (1,'Alice')");

      // Select
      ResultSet rs = st.executeQuery("SELECT * FROM users");
      while (rs.next())
        System.out.println(rs.getString("name"));
    }
  }
}

07 / Conclusion

When to Use SQLite

SQLite isn't trying to replace PostgreSQL — it's solving a different problem entirely.

✅ Use SQLite when…

Building a mobile or desktop app, creating prototypes, working with embedded systems, storing local app config/cache, or needing a self-contained database with no server setup.

❌ Avoid SQLite when…

Building a web app with many concurrent users, needing network-accessible databases, handling terabytes of data, requiring fine-grained user permissions, or high-frequency writes.

🔄 Alternatives

PostgreSQL — advanced features, full concurrency. MySQL — web apps, high traffic. MongoDB — flexible JSON documents. DuckDB — analytical queries on large data.

📝 Key Takeaways

  • SQLite is a self-contained, serverless, zero-config SQL database stored in a single file — ideal for embedded use.
  • It is pre-installed in Python, Android, iOS, and most operating systems — truly ubiquitous.
  • Supports full CRUD operations (CREATE, READ, UPDATE, DELETE) via standard SQL syntax.
  • Perfect for mobile apps, desktop apps, local data storage, prototypes, and offline-first applications.
  • Not suited for multi-user web applications, high-concurrency write scenarios, or large-scale data warehousing.
  • With 1B+ deployments, it is arguably the most widely deployed database engine in the world.
